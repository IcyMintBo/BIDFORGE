## 2. 当前流程链路

BIDFORGE 当前写作与扩写流程应理解为逐级放大的验证链路：

```text
精简小节
→ 小节级写作验证
→ 扩写大纲
→ 局部扩写
→ 多子节连续扩写
→ 单小点一万字级二次扩写
→ 半章节扩写
→ 正式章节扩写
```

各阶段定位如下：

| 阶段 | 已验证 Case | 核心问题 | 解决目标 |
|---|---|---|---|
| 小节级写作验证 | Case 001–006 | 精简小节如何写准 | 正式口吻、事实边界、服务边界、不过度承诺 |
| 局部扩写 | Case 007 | 精简小节如何进入扩写流程 | 扩写大纲、局部子节扩写、正文化检查、角色分离 |
| 多子节连续扩写 | Case 008 | 多个小点连续扩写时是否重复和变钝 | 跨节去重、节奏控制、成果落点压缩 |
| 单小点一万字级二次扩写 | Case 009 待定 | 一个小点能否单独扩展到长篇幅 | 内部结构拆分、信息密度、避免车轱辘 |
| 半章节扩写 | 后续待定 | 多个一万字级小点能否组合 | 半章节结构、篇章节奏、跨模块衔接 |
| 正式章节扩写 | 后续待定 | 是否具备真实章节产出条件 | 接近正式标书章节级输出 |

## 3. 小节级写作阶段理解确认

Case 001–006 主要验证的是：

- 如何写一个相对精简的小节；
- 如何让正文不像 AI 写作说明；
- 如何控制标书口吻；
- 如何避免空泛、越界、过度承诺；
- 如何形成 Candidate Writing Skill v0.3。

这个阶段解决的是：

```text
精简小节怎么写准
```

Candidate Writing Skill v0.3 的核心规则包括：

- Rule 001：写作重心实化规则；
- Rule 002：投标主体与落实路径规则；
- Rule 004：分析框架显性化控制规则。

该阶段不解决长篇扩写的全部问题。它主要保证短小节的正确性、正式口吻、事实边界和服务边界。

## 4. 第一次扩写阶段理解确认

Case 007 主要验证的是：

- 从一个精简小节出发；
- 先生成扩写大纲；
- 再抽样扩写 1–2 个子节；
- 检查扩写是否注水；
- 检查是否露出写作指导语；
- 检查是否有规则语言残留；
- 验证 `formalization_check` 是否有效；
- 验证 Writer / Reviewer / Rewriter / Auditor 角色分离是否必要。

这个阶段解决的是：

```text
精简小节如何进入扩写流程
```

Case 007 暴露出：

- 写作指导语外露；
- 规则语言正文化不足；
- 同上下文自审漏检；
- 需要 formalization_check；
- 需要 Writer / Reviewer / Rewriter / Auditor 角色分离。

这些内容已进入 Candidate Expansion Skill v0.1 的候选能力。

## 5. 多子节连续扩写阶段理解确认

- 检查连续阅读时是否厚重、重复、节奏偏钝。


- 需要 Candidate Expansion Rule 003：连续子节去重与节奏控制规则。

## 7. Case 009 的正确目标

Case 009 是：

```text
单小点一万字级扩写测试
```

目标是：

- 一个小点扩到 8000–12000 字；
- 不是多个小点合计 1 万字；
- 不是直接写 5 万字；
- 不是重写完整标书；
- 重点验证不车轱辘、不注水、不重复、不越界。

Case 009 的任务应是：

1. 从 `human_edit.md` 中选择一个已人工确认的小点；
2. 将该小点保存为 `source_subsection.md`；
3. 对该小点内部进行二次拆分；
4. 让每个内部模块承担不同功能；
5. 将该小点扩展到 8000–12000 字左右；
6. 检查长篇内部结构是否仍能保持信息密度和正式标书口吻。

Case 009 的目标不是：

```text
把 3–7 合起来写到 1 万字
```

而是：

```text
验证 3、4、5、6、7 这类每一个小点，是否都能单独扩写到 1 万字左右
```

## 8. Case 009 应验证的核心风险

Case 009 应重点验证以下风险：

- 车轱辘话；
- 同一观点反复换说法；
- 成果落点重复；
- 边界说明重复；
- 安全表达重复；
- 无依据事实；
- 过度承诺；
- 服务范围扩大；
- 写作指导语外露；
- 规则语言正文化不足；
- 长篇结构松散。

此外，还应继续保留：

- formalization_check；
- cross_section_review 或单小点内部结构 review；
- Writer / Reviewer / Rewriter / Auditor 角色分离；
- Human Review 优先级高于机器自审。
