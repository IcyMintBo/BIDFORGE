# BIDFORGE 项目事实与通用规则隔离机制

用途：防止 BIDFORGE 把某个项目的事实误学成通用规则。

## 内容类型与存放位置

| 内容类型 | 示例 | 存放位置 | 能否进入 Production Skill | 处理要求 |
|---|---|---|---|---|
| 项目事实 | 项目名称、建设地点、建筑面积、工期、建设内容 | 当前项目事实库 | 否 | 只服务当前项目，必须有来源 |
| 通用规则 | 无依据不得写具体面积；评分点必须映射章节 | `knowledge/` 和 Production Skill | 可以 | 必须可执行、可审查、去项目化 |
| 项目类型规则 | 产业园常关注物流组织；学校关注教学流线 | 项目类型规则库 | 可以，但需标签 | 必须标注适用项目类型 |
| 表达模式 | 审慎表达、服务边界表达、评分响应表达 | 表达库 | 可以 | 去除项目事实后使用 |
| 禁止模式 | 过度承诺、外项目痕迹、施工口径混入设计服务 | 禁止模式库 | 可以 | 保留触发特征和检测方法 |
| 测试样例 | 某次回放中的原稿、修正版、审查结果 | `lab_cases/` | 否 | 只能用于验证，不进入规则正文 |
| 候选规则 | 尚未审计的规则草案 | Candidate Skill | 否 | 经 Auditor 后才可能进入 Production |

## Production Skill 准入

可以进入 Production Skill 的内容必须满足：

- 不包含具体项目事实。
- 有明确来源。
- 有触发特征。
- 有处理动作或改写方向。
- 可被审查 Skill 检测。
- 不与招标文件优先原则冲突。
- 已通过小章节回放。
- 已经过人工确认。

## 只能进入项目事实库

- 项目名称。
- 招标人。
- 建设地点。
- 建设规模。
- 投资额。
- 工期。
- 建设内容。
- 专业范围。
- 具体图纸事实。
- 当前项目评分办法中的具体分值和条款。

## 只能作为测试样例

- 某次模型原稿。
- 某次人工改稿。
- 某个项目的风险审查报告。
- 某个项目中的错误句子。
- 某个项目的正确改写句子。

这些内容可以帮助提取规则，但不能原样进入 Production Skill。

## 需要项目类型标签

- 产业园类关注点。
- 学校类关注点。
- 市政道路类关注点。
- 公共建筑类关注点。
- 数据中心类关注点。
- 厂房类关注点。

项目类型规则离开适用类型后不得自动调用。

## 必须经过 Auditor

- 从参考标书提取的表达模式。
- 从反面教材提取的禁止模式。
- 从人工修改中提取的新文风规则。
- 从小章节回放中发现的新审查规则。
- 任何准备进入 Production Skill 的候选规则。

## 污染防护

- 参考标书不是事实来源。
- 反面教材不是整篇负样本。
- 人工改写句子不是天然通用规则。
- Candidate Skill 不是 Production Skill。
- 回放测试通过一次不等于永久有效。
