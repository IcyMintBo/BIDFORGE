# BIDFORGE 正式标书文风与措辞规则库

用途：控制 BIDFORGE 生成内容像正式投标文件正文，而不是 AI 写作说明、章节提纲、审查意见或自我防御说明。

## 规则表

| 规则 | 触发特征 | 为什么不像正式标书 | 推荐改写方向 | 审查方法 | 是否必须修改 |
|---|---|---|---|---|---|
| 禁止章节说明感 | 出现 `本节按照`、`本章主要介绍`、`以下从`、`本部分拟`、`后续章节将` 等导读词 | 像目录导读或写作说明，没有直接形成项目判断 | 改为项目理解、设计判断、工作控制逻辑 | 检查句子主语是否为章节/下文，谓语是否为介绍、说明、展开、阐述 | 是 |
| 禁止写作任务感 | 出现 `本章节需要`、`此处应体现`、`本段应说明`、`工程概述部分应` | 在指导作者怎么写，不是投标正文 | 改为项目事实重要性、设计控制要求、成果一致性要求 | 检查是否存在“章节/段落 + 应/需要 + 写作动作” | 是 |
| 禁止自我防御感 | 出现 `避免废标风险`、`不能写没有依据的内容`、`防止扩大责任`、`坚持正式稳妥口径` | 暴露内部风控逻辑，像审查意见或系统提示 | 改为服务边界、资料依据、程序衔接和成果复核表达 | 扫描内部风控词组 | 是 |
| 控制机械重复 | 连续句子以相同主语开头，连续重复 `应围绕`、`应结合`、`进一步`、`有效`、`全面`、`系统` | 像模板堆叠，缺少自然成文质量 | 调整句式功能，让相邻句分别承担事实、判断、措施、成果 | 统计段落句首结构和高频控制词 | 视程度；严重时必须修改 |
| 清理 AI 痕迹和排版残留 | Markdown 标记、错位引号、多余空格、孤立标点、半截短语、异常符号组合、中英文标点混乱 | 影响正式交付质量，暴露生成痕迹 | 清除残留，恢复正式中文正文标点和完整句子 | 扫描符号、空格、半截词和 Markdown | 是 |
| 禁止后续章节导读 | 出现 `后续章节将围绕`、`下文将进一步说明`、`后文将` | 像论文或报告导语，不构成当前章节实质内容 | 改为该判断对设计、实施和成果的支撑作用 | 扫描后续章节、下文、后文等导读词 | 通常是 |
| 从“怎么写”转为“项目怎么理解、怎么控制、怎么落实” | 出现 `写作`、`口径`、`表达`、`资料不足不能写`、`需要体现` | 暴露内部写作逻辑 | 转为项目理解、设计判断、技术控制、服务边界、成果落实、程序衔接、风险复核 | 检查是否把内部过程放进正文 | 是 |

## 推荐转换方向

| 内部写作逻辑 | 正式正文方向 |
|---|---|
| 怎么组织章节 | 项目理解、设计判断 |
| 为什么不能乱写 | 服务边界、成果依据 |
| 如何避免风险 | 程序衔接、技术复核 |
| 如何响应评分点 | 质量控制、进度控制、专业协同 |
| 资料不足 | 需进一步确认、结合后续审查、在深化阶段复核 |

## 机械重复检查

- 同一自然段内，同类句式不得连续重复超过 2 次。
- 同一小节内，收束句必须推进语义，不能只是重复标题。
- 高频词 `应`、`需`、`可`、`建议`、`通过`、`围绕`、`结合`、`落实` 需要交替使用。
- 如果连续两段结构高度相似，应重写其中一段。

## 输出前清理清单

- 无 Markdown 残留。
- 无错位引号。
- 无孤立标点。
- 无异常空格。
- 无半截短语。
- 无复制粘贴残留。
- 中文正文标点统一。
- 无明显 AI 自我解释。
